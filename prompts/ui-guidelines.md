# UI Guidelines

## Typography
- **Headings:** Use **Space Grotesk** (700-900 weight). It provides a geometric, futuristic feel appropriate for an electronics engineer.
- **Body:** Use **Inter** (400-500 weight). Optimized for readability on technical documentation sites.

## Color Palette
- **Background:** Primary Background: `#050A14` (Deep Deep Navy).
- **Accent 1:** `#00E5FF` (Electric Cyan) — for primary buttons, highlights, and icons.
- **Accent 2:** `#64748B` (Steel Gray) — for secondary text and borders.
- **Gradients:** Use subtle gradients (Deep Navy to Slightly Lighter Navy) for card backgrounds to add depth.

## Iconography
- **Set:** Lucide Icons.
- **Style:** Stroke weight 2px. Use Electric Cyan for active states or emphasis. For general listing, use Steel Gray.

## Motion & Interactivity
- **Hover States:** Links and buttons should have a slight glow effect (cyan shadow) and scale up by 1.02x.
- **Page Transitions:** Smooth fade-ins or industrial "slide" animations.
- **Data Visualization:** Use animated charts (Recharts/Chart.js) to show real-time sensor data simulation on project pages.
