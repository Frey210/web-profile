# Project Context Template

## Overview
When generating content or code for a specific project, refer to the corresponding subfolder in `projects/`. Use the `overview.md` for problem/solution context and the `README.md` for quick facts.

## System Details
Consult the `architecture.md` and `tech-stack.md` files within the project folder. Ensure any descriptions of the system match the documented data flow and component breakdown.

## Achievement Narrative
Always frame the project based on the `contribution.md` and `results.md` files. Focus on Fariz's specific role (e.g., Lead Embedded Engineer) and the tangible outcomes (e.g., "Deployed to production and used by real farmers").

## Link Integration
Use the `links.md` file in the project folder to retrieve valid GitHub repository URLs and live deployment links. Never guess a URL; always use the documented source.
