# Content Strategy

## Voice and Tone

**Voice:** Authoritative, precise, technically grounded — yet human and approachable.

Fariz is not writing a corporate brochure. He is a practitioner addressing peers, recruiters, and potential collaborators directly. The tone should feel like a confident engineer explaining their work to someone technically sophisticated, not overselling or using empty buzzwords.

**Tone Guidelines:**
- **Professional, not corporate** — No "synergy" or "leverage". Use plain, precise language.
- **Confident, not arrogant** — Let results speak: "The system monitors X parameters in real-time, deployed to production, used by Y clients."
- **Specific, not vague** — Always cite actual technologies, metrics, and outcomes. "Built a FastAPI backend" not "developed backend infrastructure."
- **Narrative-driven** — Every project has a problem and a solution. Frame the story that way.
- **First person, active voice** — "I designed the system architecture" not "The system architecture was designed."

**Vocabulary to use:** Build, Design, Architect, Implement, Deploy, Commission, Monitor, Integrate, Lead, Deliver
**Vocabulary to avoid:** Passionate, Dynamic, Synergy, Leverage (as a verb), Guru, Ninja, Rockstar

## Content Inventory

### Homepage Copy Requirements:
- Hero headline (1 line, punchy)
- Hero subheadline (2–3 lines, positioning statement)
- "About me" blurb (3–4 sentences for the homepage preview)
- 3 Featured project cards (title, 1-sentence description, tech tags, link)
- Stats bar (years experience, projects, awards, publications)
- Skills section intro (1 sentence)
- CTA buttons (2: "See My Projects", "Get In Touch")

### About Page Copy Requirements:
- Full biography (~400 words)
- Career timeline entries (3 roles: Angkasa Pura I, PT MTC / Aerasea, AirNav Internship)
- Education entry (Universitas Hasanuddin, GPA, specialization)
- Certifications list
- Personal interests section (short, humanizing)

### Project Page (per project) Requirements:
- Project name, year, category tag
- 1-sentence summary (for card/meta)
- Problem statement (~100 words)
- Solution description (~150 words)
- Architecture section (diagram or bullet breakdown)
- Tech stack (badges/tags)
- "My Role & Contribution" (~100 words)
- Results & outcomes (bullet points with numbers where available)
- Links (GitHub repos, live URL, documentation)

### Research Page Requirements:
- Paper citation (APA/IEEE format)
- Abstract/summary (~150 words)
- Links (journal, Semantic Scholar, repository)
- PKM project summaries (same structure)
- SINTA / Google Scholar profile links

### Achievements Page:
- Awards (name, year, issuer, brief context)
- Certifications (name, platform, date, verification link)
- Academic honors (GPA, funded researcher status)

### Contact Page:
- Short invitation copy (~50 words)
- Contact form
- Professional links (LinkedIn, GitHub, email)
