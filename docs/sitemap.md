## Page Hierarchies & Information Flow

### Homepage (`/`)
**Purpose:** First impression, brand statement, navigation hub
**Sections:**
- Hero — Name, title, location, animated tagline ("Builder at the Edge")
- Brief profile summary (2–3 sentences)
- Featured projects (3 highlighted projects: Aquanotes, Ergoquipt, Airport)
- Quick stats (Years of experience, Projects shipped, Awards, Publications)
- Skills overview (visual tech stack grid)
- CTA: "View Projects" → `/projects` and "Contact Me" → `/contact`

### About (`/about`)
**Purpose:** Full professional profile
**Sections:**
- Biography
- Career timeline (dual-track: Airport QA + Startup CTO)
- Education (Universitas Hasanuddin)
- Certifications (Dicoding, Cisco Networking Basics)
- Personal interests

### Projects (`/projects`)
**Purpose:** Portfolio of all major technical work
**Filter options:** IoT, Aviation, Robotics, Software, Research
**Layout:** Card grid with: project name, year, tech badges, one-line summary, links

### Individual Project Pages (`/projects/[slug]`)
**Sections:**
- Problem statement
- Solution overview
- Architecture diagram
- Tech stack used
- Fariz's specific contribution
- Results and outcomes
- Links (GitHub, live demo, documentation)

### Research (`/research`)
**Purpose:** Academic credibility — demonstrates Fariz is not only a practitioner but a researcher
**Sections:**
- Published papers (with abstracts, links)
- Research projects (PKM grants)
- Academic profile links (SINTA, Google Scholar, Semantic Scholar)

### Achievements (`/achievements`)
**Purpose:** Social proof — awards, grants, certifications
**Layout:** Timeline or grid format

### Contact (`/contact`)
**Purpose:** Convert visitors to leads/connections
**Sections:**
- Contact form (name, email, message, type of inquiry)
- Direct email
- LinkedIn, GitHub links
- WhatsApp (optional — business context)
- Company link (aerasea.id)

## URL Strategy

- All project slugs use kebab-case
- No trailing slashes
- Canonical URLs set for SSO/SEO
- Sitemap.xml auto-generated by Next.js sitemap plugin
- robots.txt: allow all, point to sitemap.xml
